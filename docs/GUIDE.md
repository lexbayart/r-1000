# Р · нарезка — Полное руководство

> Пошаговое описание всех функций приложения. Для новичков и опытных пользователей.

---

## 🖱️ Начало работы

### Загрузка видео

Есть три способа загрузить видео:

1. **Перетащить файлы** — перетащите один или несколько видеофайлов прямо на окно браузера. Поддерживаемые форматы: MP4, MOV, WebM, MKV, AVI, M4V, 3GP, WMV, FLV, OGV.
2. **Открыть папку** — нажмите кнопку «📁 или открой папку целиком» на стартовом экране. Браузер спросит разрешение на доступ к папке один раз — все видео из неё загрузятся автоматически. (Доступно в Chrome/Edge.)
3. **Выбрать через диалог** — кликните по области загрузки, откроется стандартный диалог выбора файлов.

При загрузке нескольких файлов они появляются на экране по одному с небольшой задержкой — коллаж «растёт» плиткой за плиткой.

### Коллаж видео

Загруженные видео отображаются плитками на тёмном фоне. Плитки автоматически перестраиваются при изменении размера окна.

- **Клик по плитке** — переключает активное видео в центральный плеер и показывает его звуковую дорожку.
- **Перетаскивание плитки** — зажмите и переместите плитку, чтобы изменить порядок в коллаже.
- **Кнопка ✕** — появляется при наведении на плитку; удаляет видео из сессии. Если по этому видео были отметки, они сохранятся в файл.
- **Бейджи ✅/❌** — показываются на плитке, если из этого видео что-то было вырезано. Зелёный — количество правильных, красный — неправильных.

## 🧠 Нейросетевое распознавание «Р»

При загрузке видео нейросеть автоматически начинает сканировать его звуковую дорожку. В центре экрана появляется индикатор с прогрессом (процент, ETA, статус «загрузка модели» / «сканирую»).

Модель Edge Impulse работает локально в браузере (WebAssembly). Аудио из видео декодируется в моно 16 кГц, разбивается на окна, каждое окно классифицируется. Если уверенность модели превышает порог — на волне появляется зелёная зона.

### Настройки нейросети (кнопка 🧠 на правой панели)

Панель настроек нейросети открывается кнопкой 🧠 на правой панели. Она «растёт» из панели волны — выглядит как продолжение звуковой дорожки.

**Пресеты (левая колонка):**
- **★ А: по умолчанию** — порог 0.60, интервал 50 мс, масштаб 48000. Базовый вариант.
- **Б: норм + низкий порог** — порог 0.62, интервал 50 мс, нормализация пика до 16000. Для записей с неравномерной громкостью.
- **В: норм умеренная** — порог 0.62, интервал 100 мс, нормализация до 20000. Баланс между скоростью и точностью.
- **Г: комбо агрессивное** — порог 0.58, интервал 50 мс, масштаб 48000, нормализация до 16000. Ловит больше, но может чаще ошибаться.

**Ручные настройки (правая колонка):**
- **Целевая метка** — какую метку из выдачи модели считать «то, что ищем». Смена требует пересчёта дорожки.
- **Порог «Р»** (0.20–0.95) — ниже: ловит больше, но чаще ошибается. Выше: строже, пропускает тихие.
- **Интервал** (50–500 мс) — шаг окна анализа. Меньше: точнее, но дольше считает.
- **Масштаб ×** (8000–65536) — коэффициент float32→int16. Стандарт: 32768. Увеличьте, если модель «не слышит» тихие звуки.
- **Нормализация пика** — чекбокс. Если включена: окна громче «Мин. пика» приводятся к «Целевому пику».
- **Мин. пик** (100–5000) — окна тише этого значения нормализация не трогает.
- **Целевой пик** (4000–32000) — к чему подтягивается пик громких окон.

Изменение порога — мгновенная перекраска зон. Изменение интервала, масштаба или нормализации — пересчёт всей дорожки с нуля (видите, как зоны перестраиваются в реальном времени).

## 📊 Звуковая дорожка и разметка

Внизу экрана — панель с волной (waveform). На ней отображаются:
- **Зелёные зоны** — нейросеть считает, что здесь есть «Р» (уверенность выше порога).
- **Жёлтая рамка** — текущее окно анализа (1 секунра по умолчанию). Перемещается по клику на волну.
- **Линия воспроизведения** — зелёная вертикальная линия, показывает текущую позицию.
- **Линейка времени** — под волной, с делениями в секундах.
- **Пунктирные рамки** — метки из прошлого прогона (если файл уже размечался ранее).
- **Сплошные рамки** — метки, поставленные в текущей сессии.

### Навигация по волне

- **Клик на волну** — перемещает жёлтое окно в точку клика.
- **Колёсико мыши** — масштабирование (зум в/из).
- **Перетаскивание** — прокрутка волны влево-вправо.
- **Ручки жёлтой рамки** — тяните за края, чтобы сузить/расширить окно (от 0.05 до 1.0 секунды).

### Проверка зон

1. Наведите жёлтое окно на зону (клик на волну или Tab).
2. Нажмите ▶️ (или Пробел) — проиграется этот кусок.
3. Поставьте ✅ (правильное произношение) или ❌ (неправильное).
4. Метка появится на волне и добавится в соответствующий архив.

Если на одну и ту же секунду уже есть метка в другой корзине — она автоматически перенесётся.

### Удаление меток

Наведите на любую метку (сплошную или пунктирную) — появится крестик ✕. Клик по крестику удалит только эту конкретную метку, не затрагивая остальные.

## ⌨️ Горячие клавиши

| Клавиша | Действие |
|---------|----------|
| `Space` | Воспроизведение / пауза |
| `←` `→` | Сдвинуть окно на 0.1 с влево/вправо |
| `Enter` | ✅ Сохранить как правильное произношение |
| `Backspace` | ❌ Сохранить как неправильное произношение |
| `P` | Прослушать текущий кусок (зацикленное воспроизведение окна) |
| `Tab` | Перейти к следующей зоне «Р» |
| `Shift+Tab` | Перейти к предыдущей зоне «Р» |

## 📦 Экспорт нарезок

На правой панели две кнопки архивов:

- **📦✅** — скачать все «правильные» нарезки zip-архивом.
- **📦❌** — скачать все «неправильные» нарезки zip-архивом.

Каждая нарезка — отдельный MP4-файл внутри архива, названный по исходному видео. Счётчики на кнопках показывают текущее количество записей в каждой корзине.

## 💾 Сохранение метаданных в файл

Каждая отметка ✅/❌ автоматически вшивается в исходный видеофайл через 1.2 секунды после последнего действия (debounce). Механизм:

- В конец MP4-файла дописывается специальный бокс с типом `RcfA`, содержащий JSON-список всех меток.
- Плееры и парсеры, не знающие этот тип бокса, просто игнорируют его — файл остаётся полностью воспроизводимым.
- При повторном открытии того же файла предыдущая разметка загружается автоматически и отображается пунктирными рамками.
- Если в текущей сессии вы переотметили момент — новая метка «перекрывает» старую.

**Важно:** для записи в файл нужно, чтобы браузер дал разрешение на запись. При загрузке через «открыть папку» разрешение запрашивается один раз на всю папку. При перетаскивании файлов — браузер спросит отдельно для каждого файла при первой записи. Если файл открыт через `<input>` (классический диалог выбора), запись в него невозможна — метки сохранятся только в сессии.

При закрытии вкладки или переключении приложения несохранённые метки записываются немедленно.

## ⚙️ Настройки (кнопка ⚙️)

- **Амплитуда волны** — ползунок усиления визуализации (×1–×6). Не влияет на работу нейросети, только на отображение.
- **📋 Скопировать логи** — копирует в буфер обмена диагностические логи сессии (полезно для отправки автору при багах). Логи сохраняются между перезагрузками страницы.

## ⚠️ Известные ограничения

- **Только Chrome/Edge** — FileSystem Access API (запись в файл, открытие папки) работает только в Chromium-браузерах. В Safari и Firefox доступна только загрузка файлов и просмотр.
- **Первое сканирование — медленное** — нейросеть обрабатывает аудио по 16 кГц моно, шаг 50–500 мс. Длинное видео (30+ минут) может занять несколько минут.
- **Модель — не распознавание речи** — это детектор одного звука. Модель не понимает слова, контекст, интонацию. Она реагирует на акустический паттерн, похожий на «Р».
- **Краш декодера** — некоторые форматы (например, MKV с определённым кодеком) могут вызвать падение вкладки при декодировании аудио. Такие файлы автоматически добавляются в чёрный список и обрабатываются медленным, но безопасным методом.
- **Один HTML-файл** — весь код в одном файле. Для работы на GitHub Pages нужны также `edge-impulse-standalone.js` и `.wasm` в той же папке.
